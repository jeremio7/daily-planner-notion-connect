## 1. Notion Integration 생성

### 1-1. Notion Integrations 페이지 접속
- https://www.notion.so/my-integrations 에 접속
- **"새 API 통합"** (또는 "New integration") 클릭

### 1-2. Integration 설정
- **이름**: `Daily Planner` (원하는 이름)
- **유형**: Internal integration
- **기능**:
  - 콘텐츠 읽기 (Read content) ✅
  - 콘텐츠 업데이트 (Update content) ✅
  - 콘텐츠 삽입 (Insert content) ✅
- **제출** 클릭

### 1-3. API 키 복사
- 생성 후 표시되는 **Internal Integration Secret** 복사
- `ntn_` 또는 `secret_`로 시작하는 긴 문자열
- 이 키를 안전하게 보관하세요

## 2. Notion 데이터베이스 준비

### 방법 A: 앱에서 자동 생성 (권장)

앱의 Notion 연동 패널에서 자동으로 DB를 생성할 수 있습니다.
→ [3단계](#3-앱에서-연결-설정)에서 설명

### 방법 B: Notion에서 직접 생성

Notion에서 직접 데이터베이스를 만들 경우 아래 속성(컬럼)을 정확히 만들어야 합니다:

| 속성 이름 | 타입 | 설명 |
|----------|------|------|
| **할일** | Title | 할일 텍스트 (필수, 기본 제목 컬럼) |
| **ID** | Text (rich_text) | 사용자 구분용 (user.name) |
| **날짜** | Date | 계획 날짜 (YYYY-MM-DD) |
| **시작시간** | Text (rich_text) | HH:MM 형식 (예: 09:00) |
| **종료시간** | Text (rich_text) | HH:MM 형식 (예: 10:00) |
| **완료** | Checkbox | 완료 여부 |
| **별점** | Number | 별점 (0~5) |

> 속성 이름은 정확히 한글로 입력해야 합니다 (대소문자, 띄어쓰기 주의).

### 2-1. Integration 연결 (중요!)

Notion 데이터베이스(또는 상위 페이지)에 Integration을 연결해야 합니다:

1. Notion에서 해당 데이터베이스 또는 페이지를 엽니다
2. 우측 상단 **⋯** (점 세개) 메뉴 클릭
3. **"연결"** 또는 **"Connections"** 선택
4. 검색창에서 **"Daily Planner"** (1단계에서 만든 이름) 선택
5. **"확인"** 클릭

> ⚠️ 이 단계를 빠뜨리면 API가 데이터베이스에 접근할 수 없어 "Could not find database" 오류가 발생합니다.

## 3. 앱에서 연결 설정

### 3-1. Notion 연동 패널 열기

앱 하단의 **"Notion 연동 ▼"** 을 클릭하여 패널을 펼칩니다.

### 3-2. API 키 입력

- **"API Key 입력"** 칸에 1단계에서 복사한 키를 붙여넣기
- **"연결"** 버튼 클릭
- 성공하면 ✅ 표시와 함께 봇 이름이 표시됩니다

### 3-3. 데이터베이스 선택

연결 성공 후 두 가지 옵션이 표시됩니다:

#### 옵션 A: 기존 DB 선택
- **"기존 DB 선택"** 드롭다운에서 원하는 데이터베이스 선택
- Integration이 연결된 DB만 목록에 표시됩니다

#### 옵션 B: 새 DB 생성
- **"페이지 선택"** 드롭다운에서 DB를 만들 상위 페이지 선택
- **"DB 생성"** 버튼 클릭
- 자동으로 올바른 속성 구조의 "Daily Planner" DB가 생성됩니다

## 4. 내보내기/가져오기 사용법

DB가 연결되면 4개의 버튼이 활성화됩니다:

### 내보내기 (App → Notion)

| 버튼 | 동작 | 설명 |
|------|------|------|
| **내보내기** | 추가 (append) | 현재 날짜의 할일/계획을 Notion에 추가합니다. 기존 Notion 데이터는 유지되며, 중복이 생길 수 있습니다. |
| **새로출력** | 교체 (replace) | 현재 날짜 + 나의 ID에 해당하는 기존 Notion 데이터를 삭제한 후, 현재 데이터를 새로 내보냅니다. |

### 가져오기 (Notion → App)

| 버튼 | 동작 | 설명 |
|------|------|------|
| **가져오기** | 추가 (append) | Notion의 해당 날짜 데이터를 현재 앱 데이터에 추가합니다. 기존 로컬 데이터는 유지되며, 중복이 생길 수 있습니다. |
| **새로입력** | 교체 (replace) | 확인 대화상자 후, 로컬 데이터를 Notion 데이터로 완전히 교체합니다. |

### ID 기반 사용자 분리

- 모든 내보내기 데이터에는 사용자의 **이름(name)** 이 `ID` 속성에 자동으로 저장됩니다
- **새로출력**: 나의 ID에 해당하는 데이터만 삭제 후 교체 (다른 사용자 데이터 영향 없음)
- **가져오기/새로입력**: 나의 ID에 해당하는 데이터만 가져옴 (없으면 날짜 기준으로 전체 가져옴)
- 같은 Notion DB를 여러 사용자가 안전하게 공유할 수 있습니다

## 5. 환경변수 설정 (선택)

서버 환경변수를 설정하면 앱에서 API 키와 DB ID를 입력하지 않아도 자동 연결됩니다.

### server/.env 파일

```env
# Notion API 키
NOTION_API_KEY=ntn_xxxxxxxxxxxxxxxxxxxxx

# Notion 데이터베이스 ID
NOTION_DB_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

### Render 배포 시

Render 대시보드 → Environment → 환경변수 추가:
- `NOTION_API_KEY` = API 키 값
- `NOTION_DB_ID` = 데이터베이스 ID 값

### 데이터베이스 ID 찾기

1. Notion에서 데이터베이스를 브라우저로 엽니다
2. URL 형식: `https://www.notion.so/[워크스페이스]/[DB_ID]?v=...`
3. `[DB_ID]` 부분이 32자리 영숫자 문자열입니다
4. 또는 앱에서 "기존 DB 선택" 후 개발자 도구에서 확인 가능

## 6. 문제 해결

### "API 키가 유효하지 않습니다"
- API 키를 다시 복사하여 붙여넣기 해보세요
- Integration이 활성 상태인지 확인하세요
- `ntn_` 또는 `secret_`로 시작하는 전체 키를 입력했는지 확인하세요

### "Could not find database"
- Notion에서 데이터베이스에 Integration을 연결했는지 확인하세요 (2-1단계)
- 데이터베이스 ID가 정확한지 확인하세요

### DB 검색 목록이 비어 있음
- Integration이 연결된 페이지/DB만 검색됩니다
- Notion에서 데이터베이스 → ⋯ → 연결 → Integration 추가를 해주세요

### 내보내기/가져오기 시 "인증이 필요합니다"
- 로그인 세션이 만료되었을 수 있습니다
- 로그아웃 후 다시 로그인해주세요

### 가져오기 시 데이터가 0개
- Notion DB에 해당 날짜의 데이터가 있는지 확인하세요
- '날짜' 속성이 정확한 날짜 형식(YYYY-MM-DD)인지 확인하세요
- ID 필터가 적용됩니다. 다른 사용자가 내보낸 데이터는 가져올 수 없습니다
  - 단, ID가 없는 데이터(이전 버전에서 내보낸 것)는 자동으로 가져옵니다

### "Could not find property with name or id: ID"
- 기존 DB에 'ID' 속성이 없는 경우 발생할 수 있습니다
- 앱이 자동으로 'ID' 속성을 추가하므로 다시 시도하면 해결됩니다
- 수동으로 추가하려면: Notion DB → 새 속성 → 이름: `ID`, 타입: `텍스트`
