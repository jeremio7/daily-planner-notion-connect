# Daily Planner (할일 계획표) - 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
Daily Planner - 할일 계획표

### 1.2 목적
일일 할일 목록을 작성하고, 시간대별 계획표로 배정하여 하루 일정을 체계적으로 관리하는 웹 애플리케이션

### 1.3 대상 사용자
- 하루 일정을 계획적으로 관리하고 싶은 개인 사용자
- PC와 모바일(Android/iOS) 환경에서 모두 사용 가능

### 1.4 배포 URL
- 서비스: https://daily-planner-1ioy.onrender.com
- 소스코드: https://github.com/jeremio7/daily-planner

## 3. 주요 기능

### 3.1 사용자 인증
- 이메일/비밀번호 회원가입 및 로그인
- JWT 토큰 기반 인증 (30일 유효)
- bcrypt 비밀번호 암호화 (salt rounds 12)
- 로그인 시도 제한 (5회 실패 시 15분 잠금)
- 사용자별 데이터 분리 (planner_{userId}.json)

### 3.2 할일 관리
- 할일 입력 (스페이스, 콤마로 복수 입력 가능)
- 태그 형태의 할일 목록 표시
- 할일 삭제 및 복원
- 중복 할일 하이라이트

### 3.3 시간 계획표
- 시작/종료 시간 선택 (다이얼 피커, 5분 단위)
- 할일을 시간대에 배정
- 자동 배정 기능
- 시간순 자동 정렬
- 완료 체크 및 완성도(별점) 평가
- 계획 삭제 시 할일 목록으로 복귀
- 드래그 순서 변경 (스왑)

### 3.4 시간표 (시각화)
- 24시간 원형 시계 형태의 일정 시각화
- 할일별 색상 구분
- 범례 표시
- 토글로 접기/펼치기

### 3.5 주간 통계
- 요일별 완료율 막대 그래프
- 주간 평균 완료율
- 평균 별점
- 오늘 날짜 하이라이트
- 토글로 접기/펼치기

### 3.6 푸시 알림
- 스케줄 시작 10분 전, 5분 전, 정시 총 3회 알림
- 서버 사이드 Web Push API (브라우저 꺼져도 수신)
- VAPID 키 기반 보안 구독
- 알림 ON/OFF 토글 (기본값 OFF, 로그인 시 초기화)
- 사용자별 타임존 자동 감지 및 적용
- 만료된 구독 자동 정리

### 3.7 날짜 관리
- 날짜 이동 (이전/다음/오늘)
- 날짜별 데이터 독립 저장

### 3.8 테마
- 라이트/다크 모드 토글
- localStorage에 설정 저장

## 5. 데이터 구조

### 5.1 사용자 (users.json)
```json
{
  "user@email.com": {
    "id": "uuid",
    "email": "user@email.com",
    "name": "사용자명",
    "password": "$2b$12$...(bcrypt hash)",
    "timezone": "Asia/Seoul",
    "createdAt": "2026-03-07T00:00:00.000Z"
  }
}
```

### 5.2 계획 데이터 (planner_{userId}.json)
```json
{
  "2026-03-07": {
    "todos": [
      { "id": "uuid", "text": "운동" }
    ],
    "schedule": [
      {
        "id": "uuid",
        "todoId": "uuid",
        "text": "운동",
        "startTime": "08:00",
        "endTime": "09:00",
        "done": false,
        "rating": 0
      }
    ],
    "deleted": [
      { "id": "uuid", "text": "삭제된 할일" }
    ]
  }
}
```

### 5.3 푸시 구독 (subscriptions.json)
```json
{
  "userId": [
    {
      "endpoint": "https://fcm.googleapis.com/...",
      "keys": { "p256dh": "...", "auth": "..." }
    }
  ]
}
```

## 6. API 명세

### 6.1 인증
| Method | URL | 설명 |
|--------|-----|------|
| POST | /api/auth/register | 회원가입 |
| POST | /api/auth/login | 로그인 |
| GET | /api/auth/me | 토큰 검증 |

### 6.2 계획 관리
| Method | URL | 설명 |
|--------|-----|------|
| GET | /api/planner/:date | 날짜별 데이터 조회 |
| POST | /api/planner/:date/todos | 할일 추가 |
| DELETE | /api/planner/:date/todos/:id | 할일 삭제 |
| POST | /api/planner/:date/schedule | 계획 배정 |
| PATCH | /api/planner/:date/schedule/:id | 완료/별점 수정 |
| DELETE | /api/planner/:date/schedule/:id | 계획 삭제 |
| POST | /api/planner/:date/schedule/swap | 계획 순서 변경 |
| POST | /api/planner/:date/auto-assign | 자동 배정 |

### 6.3 푸시 알림
| Method | URL | 설명 |
|--------|-----|------|
| GET | /api/push/vapid-key | VAPID 공개키 조회 |
| POST | /api/push/subscribe | 푸시 구독 등록 |
| POST | /api/push/unsubscribe | 푸시 구독 해제 |

## 9. 보안

- JWT 인증 (환경변수 JWT_SECRET)
- bcrypt 비밀번호 해싱 (salt 12)
- 로그인 브루트포스 방지 (5회/15분)
- Helmet 보안 헤더
- CORS 제한 (프로덕션: 동일 오리진만)
- 입력값 검증 (이메일 형식, 길이 제한)
- Express JSON body 크기 제한 (1MB)
